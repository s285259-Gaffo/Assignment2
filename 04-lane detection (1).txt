Introduction
n Lane detection is crucial for driver assistance systems and for autonomous vehicles
n It is used to compute the position of the vehicle w.r.t. to the lane lines
n The information can be used:
n To warn the driver in case of potential involuntary lane crossing
n To control the steering to maintain a desired position w.r.t. the lane lines
n To compute the radius of curvature of the lane

the distortion parameters
n The procedure is as follows:
n Print a pattern and attach it to a planar surface (normally a checkerboard pattern is used)

camera 
n Detect the feature points in the images 
n Estimate the intrinsic and extrinsic parameters using the closed-form solution as described in the 
following 
n Estimate the coefficients of the radial distortion by solving the linear least-squares
n Refine all parameters, including lens distortion parameters, by minimizing procedure

Perspective transformation
n The image obtained by the car camera 
contains a large non-road area, such as 
sky, trees on roadside, etc. 
n Processing of the image will increase the 
computational complexity and reduce the 
real-time capability
n The invalid area will interfere the lane 
information and affect the detection 
accuracy
n Only the area shown in the red box is 
the region of interest (ROI) for lane 
detection 
 
34

Perspective transformation
n Due to the imaging perspective effect, 
the traffic lanes are two non-parallel lines 
n The inverse perspective transformation 
eliminates the perspective effect and 
produces a top view image that is 
consistent with the actual situation
n In the resulting top view image, the lane 
lines are vertical and parallel to each 
other, which facilitate the identification in 
subsequent algorithms
